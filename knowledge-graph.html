📋 Complete Project Overview
ReturnRack is an AI-powered warehouse returns routing application. It uses a cascaded 4-layer prediction model to classify returned parcels to their appropriate warehouse. The system combines deterministic rules, merchant lookup, gradient boosted trees, and statistical models for accurate warehouse assignment with real-time confidence metrics.

🔗 Layer Arrangement & Details
ReturnRack uses a cascading prediction architecture where each layer progressively handles more complex patterns. Layer 1 catches high-confidence cases (~65%), Layer 2 uses merchant associations (~20%), Layer 3 applies machine learning (~15%), and Layer 4 provides statistical context for feature engineering.
⚡
Layer 1: Rule Engine
Deterministic
Coverage
65%
Accuracy
98%
Deterministic pattern matching using hardcoded and learned prefix/suffix rules. Covers 20-digit DHL/GLS codes, 11-digit patterns, international format IDs, and (Y) internal codes. No ML needed—rules are extracted directly from data analysis.
Key Patterns
Prefix rules, positional substrings, structural patterns
Functions
rulePredict(), buildRuleStats()
Output
{warehouse: str, confidence: 0-1, certain: bool}
🏪
Layer 2: Merchant Lookup
Bayesian
Coverage
20%
Accuracy
92%
Probabilistic merchant→warehouse association. Tracks high-purity merchant mappings where P(warehouse|merchant) ≫ 0.95. Falls back to Layer 3 if merchant association is ambiguous or unknown.
Strategy
Bayesian probability with confidence threshold
Functions
merchantPredict(), buildMerchantStats()
Data
merchantStats, pureMerchants, mixedMerchants
🌳
Layer 3: Gradient Boosted Trees
ML Ensemble
Coverage
15%
Type
Ensemble
Decision tree ensemble for ambiguous IDs. Learns subtle pattern dependencies through gradient boosting. Typically 30-50 trees with max depth 4-6. Uses cross-entropy loss and stratified k-fold validation for training.
Features
n-grams, positional substrings, length, Markov states
Functions
buildGBT(), predictGBT(), trainGBT(), softmaxProb()
State
gbtModel, gbtFeatureMap, gbtClasses, gbtAccuracy

Feature Extraction
Converts tracking IDs into numerical feature vectors for Layer 3 (GBT). Extracts n-grams, positional substrings, character frequencies, ID length, and Markov state transitions. Critical for ML model input preparation.

📊 Feature Types
N-grams: 1-grams, 2-grams, 3-grams
Positional: Substrings at fixed positions
Length: ID length distribution
Markov: Character transition probabilities
🔗 Relationships
Used by: Layer 3 (GBT), Layer 4 (Stats)
Part of: Data Pipeline
📈
Confidence Calculation
Computes confidence scores from deterministic rules (Layer 1) or GBT softmax output (Layer 3). Combines layer confidences, prioritizes high-confidence sources, and applies fallback logic when predictions conflict.
🧮 Formula
Primary: Conf(y) = max(RuleConf, ModelConf)
Softmax: P(y=k|x) = softmax(Σ trees f_t(φ(x)))
📊 Categories
High: confidence ≥ 0.85 (green badge)
Medium: 0.70 ≤ confidence < 0.85 (blue)
Low: 0.50 ≤ confidence < 0.70 (amber)
Very Low: confidence < 0.50 (red)
🔗 Relationships
Called by: predict() main orchestrator
Receives from: All prediction layers
📝
Session Tracking
Maintains in-session state tracking: recent scans list, warehouse counts (BER, POZ, other), merchant frequencies, and session-wide statistics. Triggers UI updates and feeds data to the Recent Scans table and Merchant Panel.
📊 State Variables
berCount: number — BER3 warehouse counts
pozCount: number — POZ warehouse counts
totalCount: number — total scanned
recentScans: array — last 50 predictions
merchantCounts: object — frequency per merchant
🔄 Scan Object
id: tracking ID
wh: predicted warehouse
conf: confidence score
merchant: merchant code
resolvedBy: which layer made decision
ts: timestamp
🔗 Relationships
Updated by: predict() after each scan
Read by: UI System for display

Recent Scans Table (UI)
Scrollable history table logging all predictions in the session. Shows TrackingID, Warehouse, Confidence, Resolved By layer, and Time. Displays up to 50 most recent scans with row highlighting on hover.

Part of: UI System module
🏪
Merchant Panel (UI)
Right sidebar panel showing merchant breakdown sorted by frequency. Displays merchant name, count of scans, and percentage of total session scans. Updates dynamically as new predictions arrive.
📊 Columns
Merchant: Merchant code or name
Count: Number of scans this session
Percentage: Count / totalCount * 100%
⚙️ Features

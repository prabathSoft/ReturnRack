Pattern 1 — ID format types
1
20-digit numeric (DHL/GLS standard)
Largest group — 1,704 records (67%). All start with 0034. The carrier/sort code at positions 8–11 is the most discriminative single feature: certain codes are 100% tied to one warehouse.
pos8-11: 7577 → BER3 96% pos8-11: 5249 → BER3 100% pos8-11: 0734 → POZ 100% pos8-11: 5320 → POZ 100%
2
11-digit numeric (GLS/national carrier)
279 records. Prefix 115 maps almost entirely to BER3. Prefix 121 splits across all three. Prefix 1155 is pure BER3.
115xxx → BER3 ~90% 1155xxx → BER3 100%
3
International format — [2 letters][digits][DE]
101 records, all end in DE (German destination). The 2-letter carrier prefix is highly predictive.
LE, CD, CB, UE → BER3 CA, UF → POZ
4
(Y) internal system codes
151 records with various subtypes. The subtype after (Y) is the key signal.
(Y)#[ALPHA] → BER3 100% (Y)[ALPHA]... → BER3 ~85% (Y)#SRDE... → POZ (Y)#[NUM] → POZ2 ~52%

Recommended model architecture
Rule engine (deterministic layer) use first
Covers ~65% of cases with near-100% accuracy using hard rules: exact ID prefix match, positions 8–11 for 20-digit IDs, carrier prefix for INTL format, (Y)# subtype. No training required, fully interpretable, zero false confidence.
~98% accuracy on cases it handles
Merchant lookup table (direct signal) use second
For the 162 pure merchants: always correct. For the 28 ambiguous merchants: pass to the ML layer. Merchant covers 68% of all rows.
~100% on pure merchants
Gradient Boosted Trees — XGBoost or LightGBM best ML choice
For the remaining ambiguous cases (~35%). Features: positional n-grams (bigrams at positions 0–12), prefix hash, suffix hash, ID length, format type, character at each position up to 14. GBT handles the mixed numeric/categorical nature of this data extremely well — better than LSTM for structured ID patterns. Fast to train, interpretable feature importances, no GPU needed.
